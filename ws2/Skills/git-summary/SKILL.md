# Git Activity Summary

Generate a structured markdown summary of all git activity in the current hawkBit repository for a user-specified period.

## Workflow

### 1. Determine the time period

Parse user input to determine period. Map to git date format:

| User says | Git --since value |
|-----------|------------------|
| "today" / "one day" / "1 day" | `1.day.ago` |
| "one week" / "1 week" / "this week" | `1.week.ago` |
| "two weeks" | `2.weeks.ago` |
| "one month" / "1 month" / "this month" | `1.month.ago` |

If ambiguous, ask. Default to one week if truly unclear.

### 2. Gather git data

Run these via Bash tool (all independent, run in parallel):

```bash
# All commits in period with stats
git log --since="<period>" --pretty=format:"%H|%h|%an|%ae|%ad|%s" --date=short --stat

# Summary stats
git log --since="<period>" --pretty=format:"%H" --shortstat

# Files changed with diff stats
git log --since="<period>" --numstat --pretty=format:"COMMIT:%h|%s"

# Authors
git log --since="<period>" --pretty=format:"%an" | sort -u

# Date range boundaries
git log --since="<period>" --pretty=format:"%ad" --date=short | sort | head -1
git log --since="<period>" --pretty=format:"%ad" --date=short | sort | tail -1
```

### 3. Categorize commits

Classify each commit by its message prefix/content into these categories:

| Category | Indicators |
|----------|-----------|
| **Features & Enhancements** | "Add", "Implement", "Introduce", "Enable", "Support", "Improve" |
| **Bug Fixes** | "Fix", "Resolve", "Correct", "Patch", "Handle" |
| **Dependencies** | "Bump", "Update.*version", "Upgrade", "Migrate.*dependency" |
| **Infrastructure & CI** | "CI", "pipeline", "workflow", "docker", "action", "build" |
| **Refactoring & Cleanup** | "Cleanup", "Refactor", "Remove", "Simplify", "Rename", "Move" |
| **Documentation** | "doc", "README", "SECURITY", "CHANGELOG", "comment" |
| **Security** | "vulnerability", "CVE", "security", "trivy", "codeql" |
| **Release** | "Release", "version", "DEPENDENCIES" |

Match case-insensitively. If commit fits multiple, pick most specific. If none match, put in "Other".

### 4. Generate markdown

Write output to `activity-summary-<start>-to-<end>.md` in the repo root.

Use this exact structure:

```markdown
# hawkBit Activity Summary

**Period:** YYYY-MM-DD to YYYY-MM-DD
**Generated:** YYYY-MM-DD

---

## Overview

| Metric | Value |
|--------|-------|
| Total Commits | X |
| Contributors | X |
| Files Changed | X |
| Lines Added | +X |
| Lines Removed | -X |

## Contributors

- **Name** — X commits

## Changes by Category

### Features & Enhancements
- `short-hash` message (#PR)

### Bug Fixes
- `short-hash` message (#PR)

### Dependencies
- `short-hash` message (#PR)

(repeat for each non-empty category)

## Daily Breakdown

### YYYY-MM-DD (Day of week)
- `short-hash` message (Author)

### YYYY-MM-DD (Day of week)
- `short-hash` message (Author)

## Most Changed Files

| File | Changes | Additions | Deletions |
|------|---------|-----------|-----------|
| path/to/file | X | +Y | -Z |

(top 10 files by total changes)

---
*Generated by git-summary skill*
```

### 5. Report to user

After writing file, tell user:
- Where file was saved
- Quick highlight of key stats (commits, contributors, top category)
- Any notable patterns (e.g., "most activity was dependency bumps" or "5 bug fixes in PostgreSQL area")

## Important

- Empty categories: omit entirely, don't show empty sections
- PR numbers: extract from commit message if present (e.g., `(#3005)`)
- If zero commits in period, say so and suggest a longer period
- Sort daily breakdown newest-first
- Sort contributors by commit count descending
- Sort "Most Changed Files" by total changes descending
